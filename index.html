Guía de meta-prompting SQL

    
Patrones para generar o modificar tablas en Supabase usando un LLM. Siempre consulta information_schema primero.

    

      

        
PASO 1 — Obtener esquema real (ejecutar primero)

        
SELECT column_name, data_type, is_nullable, column_default
FROM information_schema.columns
WHERE table_name = 'tu_tabla'
ORDER BY ordinal_position;

        
Copia el output y pégalo en el prompt de más abajo.

      

      

        
PROMPT — Crear tabla nueva

        
Ayúdame a crear una tabla en PostgreSQL (Supabase) para [descripción de lo que guarda la tabla].

Requisitos:
- La llave primaria es session_id (TEXT) porque usamos el chat.id de Telegram
- Necesito guardar [campo1, campo2, campo3...]
- Los timestamps deben ser TIMESTAMPTZ (no DATE ni TIMESTAMP)
- México es UTC-6 permanente, los timestamps se guardan en UTC
- Incluir IF NOT EXISTS
- Usar nombres en snake_case

Genera solo el SQL, sin explicación.

      

      

        
PROMPT — Modificar tabla existente

        
Tengo una tabla en Supabase con esta estructura:

[PEGA AQUÍ EL OUTPUT DEL NODO information_schema]

Necesito [agregar la columna X de tipo Y / cambiar el tipo de Z / agregar índice en W].

Genera el ALTER TABLE correspondiente. Solo SQL, sin explicación.

      

      

        
PROMPT — Generar UPSERT o INSERT

        
Tengo una tabla en Supabase con esta estructura:

[PEGA AQUÍ EL OUTPUT DEL NODO information_schema]

Necesito un INSERT que:
- Registre al usuario si no existe (usar ON CONFLICT DO UPDATE)
- Actualice ultima_interaccion = NOW() en cada mensaje
- El session_id viene de {{ $json.message.chat.id }} (Telegram)
- El campo status debe quedar en blanco si es nuevo usuario

Genera solo el SQL con las expresiones n8n en doble llave.

      

      

        
PROMPT — Generar SELECT con filtros

        
Tengo una tabla en Supabase con esta estructura:

[PEGA AQUÍ EL OUTPUT DEL NODO information_schema]

Necesito un SELECT que:
- Devuelva usuarios inactivos desde hace más de [N] horas
- La condición es: ultima_interaccion < NOW() - INTERVAL '[N] hours'
- También que recordatorio = FALSE
- Ordenados del más antiguo al más reciente

Genera solo el SQL.

      

      

        
TABLA REFERENCIA — registro_usuarios

        
CREATE TABLE registro_usuarios (
    session_id          TEXT PRIMARY KEY,      -- chat.id de Telegram (TEXT, no integer)
    primer_contacto     TIMESTAMPTZ,           -- se escribe una sola vez
    ultima_interaccion  TIMESTAMPTZ,           -- se actualiza en cada mensaje
    status              TEXT,                  -- último intent: CONVERSACION/FAQ/ACCION
    recordatorio        BOOLEAN DEFAULT FALSE, -- re-engagement enviado
    timezone            TEXT DEFAULT 'America/Mexico_City'
);

-- UPSERT por mensaje:
INSERT INTO registro_usuarios (session_id, primer_contacto, ultima_interaccion)
VALUES ('{{ $json.telegram_id }}', NOW(), NOW())
ON CONFLICT (session_id)
DO UPDATE SET ultima_interaccion = NOW();

-- SELECT EXISTS (siempre devuelve 1 fila boolean, sin Always Output Data):
SELECT EXISTS (
    SELECT 1 FROM registro_usuarios
    WHERE session_id = '{{ $json.telegram_id }}'
) AS usuario_existe;
